Display and visibility

Display is key to layout

  • Every element is a box
  • Display affects the layout of neighboring elements

Common Values

  • Display:inline;
    • Sits next to other elements
    • Takes up "just enough" width and height
  • Display:block;
    • forces a line break
    • Default take up all horizontal width and "just enough" height
    • Rules can set height and width
  • Display:Inline-block;
    • Same as inline, but accepts height and width
  • Display:none;
    • removed from page

Complementary Properities

  • Float:
    • Reposition elements to the right or left
    • Elements are aware of one another and will not overlap
    • Values: left, right
  • Clear:
    • Used to keep floating elements away
    • Values: left, right, both

Display and visibility - Part 2

Element overflow

  • What happens when you set a height/width and the content doesn't fit any longer?
  • Use overflow to determine access

Overflow:

  • Overflow:Visible; Can cause text to show up "on top" of other text
  • Overflow:hidden; Hides anything that goes beyond bounding boxThis can cause problems since if the user increases font size, they man not be able to see content
  • Overflow:scroll; Gives horizontal and vertical scrollbars
  • Overflow:auto; Adds scrollbars as needed

Other display values

  • New display properties are avilaible, but not always supported:
    • Table
    • Grid
    • Flexbox

Display:table;

  • Sometimes you want to have table-like layout without using table structure, use display:table along with display:table-cell for elements.

Div display:table-cell;

Visibility:

  • Specifies whether or not element is visible
  • Options include:
    • Visibility:visible;
    • Visibility:hidden; invisible but empty space is left
    • Visibility:collapse; (only for table elements)
